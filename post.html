GitHub Pages로 나만의
블로그 만들기

복잡한 설정 없이, 순수 HTML과 CSS만으로 애플 감성의 블로그를 만드는 방법

2026년 7월 18일 5분 읽기 #개발 #CSS #GitHub
🚀

블로그를 만들겠다고 마음먹고 나서, Jekyll이니 Hugo니 Next.js니 여러 도구들을 찾아봤다. 그런데 결국 내가 선택한 건 그냥 HTML 파일이었다.

복잡한 빌드 시스템도, npm 패키지도 없이. 그냥 메모장(이라고 쓰고 VS Code라고 읽는)에서 파일 하나 만들고, GitHub에 올리면 끝. 이게 얼마나 심플한지 한번 보여줄게요.

왜 GitHub Pages인가?

GitHub Pages는 GitHub 저장소를 그대로 웹 호스팅으로 써주는 서비스예요. 무료고, 커스텀 도메인도 붙일 수 있고, HTTPS도 자동으로 적용돼요. 블로그 용도로는 정말 완벽한 선택입니다.

"최고의 도구는 내가 실제로 쓰는 도구다." — 어느 개발자

시작하기 전에 필요한 것

딱 두 가지만 있으면 됩니다:

  • GitHub 계정
  • 텍스트 에디터 (VS Code 추천)

Step 1: 저장소 만들기

GitHub에서 새 저장소를 만들 때 이름을 [내아이디].github.io 형식으로 짓는 게 핵심이에요. 이 특별한 이름 덕분에 GitHub이 자동으로 Pages를 활성화해줍니다. 

# 예시
저장소 이름: woni.github.io
접근 URL: https://woni.github.io

Step 2: index.html 만들기

저장소를 클론한 뒤, index.html 파일을 하나 만들어요. 이 파일이 블로그의 메인 페이지가 됩니다. 

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>내 블로그</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <h1>안녕하세요!</h1>
</body>
</html>

Step 3: 배포하기

파일을 저장하고 GitHub에 push하면 끝이에요. 몇 분 안에 https://[내아이디].github.io에서 바로 확인할 수 있습니다. 

git add .
git commit -m "첫 번째 블로그 포스팅"
git push origin main

이게 전부예요. 정말로. 복잡하게 생각할 필요 없이, 지금 당장 시작해보세요. 완벽한 블로그보다 지금 존재하는 블로그가 훨씬 낫습니다. 🎉
